## 七、问题回顾

### 7.1 为什么之前的修复没有效果？

- 添加了`dev_pid: 1537`参数
- 但使用了**错误的值**（1537是中文）
- 所以还是识别中文

**第二次修复（步骤30）**：
- 修改为`dev_pid: 1737`
- 使用了**正确的值**（1737是英文）
- 现在可以识别英文了 ✅

### 7.2 为什么文字模式可以识别英文？

**文字模式**：
- 用户直接输入英文文字
- 不经过语音识别API
- 直接发送给AI对话API
- 所以不受`dev_pid`参数影响

**语音模式**：
- 用户说话 → 录音 → 语音识别API → 文字
- 语音识别API使用`dev_pid`参数判断语言
- 如果`dev_pid`错误，识别结果就错误

### 7.3 关键教训

1. **仔细查阅API文档**
   - 不要凭感觉猜测参数值
   - 必须查阅官方文档确认

2. **参数值很重要**
   - `dev_pid: 1537`和`dev_pid: 1737`只差一个数字
   - 但结果完全不同（中文 vs 英文）

3. **测试验证**
   - 修改后必须测试验证
   - 查看日志确认参数传递正确

## 八、百度语音识别API完整参数说明

### 8.1 必需参数

```typescript
interface RecognitionPayload {
  format: string;      // 音频格式：wav, pcm, amr, m4a
  rate: number;        // 采样率：8000, 16000
  dev_pid: number;     // 语言模型：1537(中文), 1737(英文), 1837(粤语)
  cuid: string;        // 客户端唯一标识
  speech: string;      // Base64编码的音频数据
  len: number;         // 音频数据字节数
}
```

### 8.2 可选参数

```typescript
interface RecognitionPayloadOptional {
  channel: number;     // 声道数：1(单声道), 2(双声道)
  token?: string;      // 访问令牌（使用API Key时不需要）
}
```

### 8.3 响应格式

**成功响应**：
```json
{
  "err_no": 0,
  "err_msg": "success",
  "corpus_no": "7136997683261261820",
  "sn": "123456789",
  "result": ["Hello, how are you?"]
}
```

**错误响应**：
```json
{
  "err_no": 3301,
  "err_msg": "音频质量过差",
  "sn": "123456789"
}
```

### 8.4 常见错误码

| err_no | 错误消息 | 说明 |
|--------|---------|------|
| 0 | success | 成功 |
| 3300 | 输入参数不正确 | 检查参数格式 |
| 3301 | 音频质量过差 | 改善录音环境 |
| 3302 | 鉴权失败 | 检查API Key |
| 3303 | 语音服务器后端问题 | 稍后重试 |
| 3304 | 用户的请求QPS超限 | 降低请求频率 |
| 3305 | 用户的日pv（日请求量）超限 | 升级配额 |

## 九、常见问题

### Q1: 为什么修改代码后还是识别中文？

**A**: 需要重新部署Edge Function并刷新浏览器。

**解决步骤**：
1. 确认代码已修改为`dev_pid: 1737`
2. 运行`npm run lint`检查代码
3. 重新部署Edge Function
4. **强制刷新浏览器**（Ctrl+F5或Cmd+Shift+R）
5. 清除浏览器缓存（可选）
6. 重新测试

### Q2: 如何确认dev_pid参数已正确传递？

**A**: 查看Supabase Edge Function日志。

**操作步骤**：
1. 进入Supabase项目后台
2. 点击"Edge Functions"
3. 点击"phone-call-dialogue"
4. 点击"Logs"标签
5. 查找"dev_pid: 1737 (英文识别 - English)"
6. 确认完整请求体中有`"dev_pid": 1737`

### Q3: 英文识别准确率还是不高怎么办？

**A**: 优化录音质量和说话方式。

**建议**：
1. **环境**：在安静的环境中录音
2. **设备**：使用质量好的麦克风
3. **距离**：靠近麦克风（10-30cm）
4. **音量**：正常音量，不要太小或太大
5. **语速**：适中，不要太快
6. **发音**：清晰，标准发音
7. **时长**：2-10秒为佳
8. **内容**：避免过于复杂的句子

### Q4: 可以同时识别中英文吗？

**A**: 不可以，必须选择一种语言模型。

**说明**：
- `dev_pid: 1537`：只识别中文
- `dev_pid: 1737`：只识别英文
- 不能同时识别两种语言
- 如需切换，需要修改参数并重新部署

### Q5: 如何切换回中文识别？

**A**: 修改`dev_pid`参数为1537。

**操作步骤**：
1. 打开`supabase/functions/phone-call-dialogue/index.ts`
2. 找到`dev_pid: 1737`
3. 改为`dev_pid: 1537`
4. 重新部署Edge Function
5. 刷新浏览器测试

### Q6: 为什么有时候还是会出现配额耗尽错误？

**A**: 语音识别插件有每日免费配额限制。

**解决方案**：
1. **等待配额重置**：每天0点（UTC+8）自动重置
2. **使用文字模式**：点击键盘图标切换
3. **升级配额**：联系平台管理员购买更多配额
4. **减少测试**：开发时优先使用文字模式
