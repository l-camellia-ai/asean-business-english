# AI电话工坊调试指南

## 问题诊断

### 已修复的问题

#### 1. Edge Function响应处理错误
**问题**：前端没有正确处理Edge Function返回的500状态码
**修复**：
- 改进了响应检查逻辑
- 先检查 `result.error`（网络错误）
- 再检查 `result.data.success`（业务逻辑错误）
- 添加详细的日志输出

#### 2. 音频转换错误处理不足
**问题**：音频解码失败时没有明确的错误提示
**修复**：
- 添加详细的转换日志
- 捕获解码错误并提供友好提示
- 记录每个转换步骤的数据大小

#### 3. 错误信息不明确
**问题**：用户看到的错误信息太笼统
**修复**：
- 区分不同类型的错误（网络、音频、识别、配置等）
- 提供针对性的解决建议
- 添加详细的控制台日志

## 测试步骤

### 步骤1：测试文字输入模式（推荐先测试）

这是最简单的测试方式，可以验证Edge Function是否正常工作。

1. 打开应用，进入AI电话工坊
2. 选择任意场景（如"商务问候"）
3. 点击"开始通话"
4. 点击"切换到文字输入"按钮
5. 在输入框中输入英语，例如：`Hello, how are you?`
6. 点击发送按钮
7. 观察是否收到AI回复

**预期结果**：
- ✅ 用户消息显示在对话区
- ✅ AI回复显示在对话区
- ✅ 可能播放语音（如果TTS成功）
- ✅ 分数增加

**如果失败**：
- 打开浏览器开发者工具（F12）
- 查看Console标签页的错误信息
- 查看Network标签页，找到 `phone-call-dialogue` 请求
- 检查请求和响应的详细信息

### 步骤2：测试语音录音模式

在文字模式测试成功后，再测试语音模式。

1. 确保已允许浏览器访问麦克风
2. 点击"切换到语音识别"按钮
3. 点击"点击录音"按钮
4. 对着麦克风说英语（至少2秒）
5. 点击"停止录音"
6. 等待处理（会显示"正在处理音频，请稍候..."）
7. 观察识别结果和AI回复

**预期结果**：
- ✅ 录音时显示时长计数
- ✅ 停止后显示"正在处理音频"
- ✅ 识别出的文字显示在对话区
- ✅ AI回复显示在对话区
- ✅ 播放AI语音

**如果失败**：
- 检查麦克风权限
- 查看Console日志，特别是：
  - "开始处理音频" - 音频大小和类型
  - "音频转换完成" - 转换后的大小
  - "Base64编码完成" - 编码长度
  - "发送请求到Edge Function" - 请求参数
  - "Edge Function响应" - 响应数据

## 常见错误及解决方案

### 错误1：Edge Function returned a non-2xx status code

**原因**：Edge Function返回了500错误
**排查**：
1. 检查Console日志中的详细错误信息
2. 可能的原因：
   - API密钥未配置或无效
   - 语音识别API失败
   - AI对话API失败
   - 语音合成API失败

**解决方案**：
- 检查Supabase Secrets中的 `INTEGRATIONS_API_KEY` 是否配置
- 使用文字模式测试，跳过语音识别步骤
- 查看Edge Function日志（如果有权限）

### 错误2：音频格式不支持

**原因**：浏览器录制的音频格式无法解码
**解决方案**：
- 使用Chrome 90+或Edge 90+浏览器
- 检查是否有其他应用占用麦克风
- 尝试刷新页面重新开始

### 错误3：录音时间太短

**原因**：录音少于2秒
**解决方案**：
- 至少说2秒以上的完整句子
- 观察录音时长提示

### 错误4：请求超时

**原因**：网络不稳定或API响应慢
**解决方案**：
- 检查网络连接
- 系统会自动重试2次
- 可以切换到文字输入模式

### 错误5：语音识别失败

**原因**：
- 环境噪音太大
- 说话不清晰
- 麦克风音量太小

**解决方案**：
- 在安静环境中录音
- 靠近麦克风说话
- 检查麦克风音量设置
- 使用文字输入模式作为替代

### 2. 检查网络请求

在Network标签页中：
1. 筛选 `phone-call-dialogue`
2. 查看请求头（Headers）
3. 查看请求体（Payload）
4. 查看响应（Response）

可以直接测试Edge Function：

```bash
curl -X POST https://your-project.supabase.co/functions/v1/phone-call-dialogue \
  -H "Authorization: Bearer YOUR_ANON_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "userText": "Hello",
    "conversationHistory": [],
    "scenarioTitle": "商务问候",
    "voiceId": "male-qn-qingse"
  }'
```

### 4. 使用文字模式调试

文字模式跳过了音频录制和转换步骤，可以快速验证：
- Edge Function是否正常
- API密钥是否有效
- AI对话是否正常
- 语音合成是否正常

## 性能优化建议

### 1. 减少对话历史长度
当前保留最近10条对话，可以根据需要调整：
```typescript
.slice(-10) // 改为 .slice(-5) 可以减少请求大小
```

### 2. 调整音频质量
如果网络不好，可以降低音频质量：
```typescript
sampleRate: 16000 // 已经是最低的了
```

### 3. 增加超时时间
如果网络慢，可以增加超时：
```typescript
setTimeout(() => reject(new Error('请求超时')), 30000) // 改为 60000
```

## 已知限制

1. **浏览器兼容性**：仅支持Chrome 90+和Edge 90+
2. **录音时长**：最短2秒，最长60秒
3. **文件大小**：最大10MB
4. **网络要求**：需要稳定的网络连接
5. **麦克风要求**：需要清晰的音频输入
