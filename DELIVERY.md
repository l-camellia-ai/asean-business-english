## 🔍 问题根源

通过详细的日志分析和代码审查，发现以下核心问题：

### 1. Edge Function响应处理错误
- **现象**：前端显示"Edge Function returned a non-2xx status code"
- **原因**：没有正确解析Supabase Functions的响应结构
- **影响**：即使Edge Function正常工作，前端也会报错

### 2. 音频转换缺少容错
- **现象**：音频解码失败时直接崩溃
- **原因**：没有捕获解码错误
- **影响**：用户无法知道具体是什么问题

### 3. 错误信息不明确
- **现象**：所有错误都显示"处理失败"
- **原因**：没有区分不同类型的错误
- **影响**：用户和开发者都难以定位问题

## ✅ 修复内容

### 1. 核心修复：Edge Function响应处理

**修复前**：
```typescript
const { data, error } = await supabase.functions.invoke(...)
if (error || !data.success) {
  throw new Error('处理请求失败');
}
```

**修复后**：
```typescript
const result = await supabase.functions.invoke(...)

// 先检查网络错误
if (result.error) {
  console.error('Edge Function调用错误:', result.error);
  throw new Error(result.error.message || '网络请求失败');
}

// 再检查业务逻辑错误
if (!result.data || !result.data.success) {
  throw new Error(result.data?.error || '服务器处理失败');
}
```

## 🔧 故障排查

### 问题1：文字模式失败

**可能原因**：
- API密钥未配置或无效
- 网络连接问题
- Edge Function未正确部署

**解决方案**：
1. 检查Supabase Secrets中的 `INTEGRATIONS_API_KEY`
2. 查看Console的详细错误信息
3. 查看Network中的请求和响应
4. 联系管理员检查API配额

### 问题2：语音模式失败

**可能原因**：
- 浏览器不支持（需要Chrome 90+或Edge 90+）
- 麦克风权限被拒绝
- 音频格式不支持
- 录音时间太短（<2秒）
- 环境噪音太大

**解决方案**：
1. 使用推荐的浏览器
2. 允许麦克风权限
3. 在安静环境中录音
4. 至少说2秒以上
5. 查看Console的详细日志
6. 如果仍然不行，使用文字模式

### 问题3：识别结果不准确

**解决方案**：
- 在安静环境中录音
- 靠近麦克风说话
- 清晰发音，语速适中
- 使用标准英语发音
- 避免方言和俚语
